# 可靠性审查清单

> 审查官可在审查报告中引用这些模式 ID，如 `> ✅ REL-01 已覆盖`。

| ID | 检查项 | 说明 |
|----|--------|------|
| REL-01 | 空值/空列表处理 | nullable 参数是否做了 `?.` 或 `?:` 兜底；列表是否判空 |
| REL-02 | 异常捕获 | 网络/DB/文件操作是否 try-catch；catch 后是否有日志或降级 |
| REL-03 | 并发安全 | 多线程读写同一变量是否同步；`@Volatile`/`synchronized`/`Atomic` 是否到位 |
| REL-04 | 生命周期安全 | view/context 使用前是否判 `isFinishing`/`isDestroyed`；`lifecycleScope` 是否正确 |
| REL-05 | 超时与重试 | 网络请求是否设置了超时；失败后是否有重试策略（含退避） |
| REL-06 | 降级兜底 | 关键依赖不可用时是否有默认值/缓存兜底，而非直接崩溃 |
| REL-07 | 数据一致性 | 写入操作是否 atomic；中途崩溃是否会留下半截数据 |
| REL-08 | 边界值 | 0、负数、空串、超长字符串、特殊字符是否测试过 |

## 审查引用方式

审查报告中：
```
## 可靠性评估
| ID | 检查项 | 结论 |
|----|--------|:---:|
| REL-01 | 空值处理 | ✅ 已覆盖 — 传入了 nullable 参数，已用 `?:` 兜底 |
| REL-02 | 异常捕获 | 🆕 新发现 — 网络请求无 try-catch，网络断开会崩溃 |
```
