# 安全审查清单

> 审查官可在审查报告中引用这些模式 ID，如 `> ✅ SEC-01 已覆盖`。

| ID | 检查项 | 说明 |
|----|--------|------|
| SEC-01 | 敏感信息泄露 | 日志/Toast/弹窗中是否有 API key、token、密码、手机号等明文 |
| SEC-02 | 本地存储安全 | SharedPreferences/MMKV/UserDefaults 是否存储了敏感数据未加密 |
| SEC-03 | 网络传输明文 | HTTP 请求中是否有明文密码/token；是否应强制 HTTPS |
| SEC-04 | 权限最小化 | 新增权限是否必要；是否在运行时动态申请而非静态声明 |
| SEC-05 | 输入验证 | 用户输入是否做了长度/格式/注入防护（SQL、XSS、路径穿越） |
| SEC-06 | 出口数据脱敏 | 日志上报、崩溃上报中是否包含用户可识别信息（PII） |
| SEC-07 | WebView 安全 | `setJavaScriptEnabled` 是否必要；`addJavascriptInterface` 是否做白名单 |

## 审查引用方式

审查报告中：
```
## 安全自审
| ID | 检查项 | 结论 |
|----|--------|:---:|
| SEC-01 | 敏感信息泄露 | ✅ 已覆盖 — 无新增日志输出 |
| SEC-03 | 网络传输 | 🆕 新发现 — API 参数含明文 userId |
```
