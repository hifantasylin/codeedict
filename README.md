# AI码律(Codeedict)

面向超级个体的标准化开发工作流。一份工作空间，管理你的所有项目。每次修 bug、加功能、做分析，方案自动归档，历史随时检索，换了电脑拷个工作空间目录就能接着干。如果说 AI 是你的副驾驶，AI码律 就是仪表盘——状态、记录、检查点，全靠它。任何语言、任何项目通用。

## 快速开始

安装：告诉 AI `请安装 AI码律/Codeedict`。安装后在 AI 编辑器中选择 **Codeedict** agent 即可使用。

> 支持 **CodeBuddy** / **VS Code (GitHub Copilot)** 双平台。

```
# 初始化项目（首次使用）
初始化 my-app
扫描 my-app      # 等价于初始化

# 修 bug / 加功能 → 自动走 Code 流程
登录页点了没反应

# 代码分析 → 自动走 Clarify 流程（分析子路径）
帮我分析下启动流程

# 梳理需求 / 新功能 → 自动走 Clarify 流程（修改子路径）
我想做一个夜间模式

# 其他命令
继续 my-app-B01-xxx    # 从断点恢复
周报                    # 生成本周报告
```

## 功能亮点

**多项目工作空间 + 文档沉淀**：一份工作空间管理多个项目，装一次用一辈子。每个项目独立持有自己的方案、历史索引——切项目不丢知识，重复问题自动检索历史方案。所有高级能力（审查官、索引、断续恢复）都建立在这个文档沉淀的基础之上。

**双审查官质量门**：方案产出后方案审查官从目标、成本、风险、长期收益、异常路径、安全自审、回滚方案七角度对立审查，不顺着思路找漏洞。代码完成后代码审查官逐项对账，偏离分三层裁决——执行层直接放行，内循环层审查官自主裁决（接受或驳回 coder 修正），只有方向性变更才上升用户。多数偏离在后台消化，用户只在真正需要拍板的决策点被打扰。

**历史任务索引**：每个任务归档时在 `archive/index.md` 自动追加一行索引（ID、标题、类型、症状关键词）。分析师定位根因前自动检索最近 12 个月的匹配条目，命中则引用历史方案。同一/同类问题复发时快速定位。

**编码任务拆解 + 断续恢复**：编码阶段先按提案拆成可独立执行的任务，写入 `tasks/<taskId>-tasks.md` 表格追踪状态（⏳/🔧/✅/⚠️）。中断后 `继续 <id>` 秒级恢复。

**自动路由 + 意图感知**：说"登录页闪退"自动走 Code 流程，说"分析下启动性能"自动走 Clarify 分析子路径，说"加个夜间模式"自动走 Clarify 修改子路径。意图模糊时会追问。

**周报生成**：`周报` 一键输出本周完成、进行中、受阻项，附任务摘要和进度。

## 对比优势

**vs 纯 prompt 工作流**：有状态机硬约束保证 AI 不跳阶段，不会出现"顺手帮你改了"的越界行为。审查官独立裁决，不靠 AI 自觉。

**vs spec-driven / plan-then-execute 模式**：不过早冻结实现方案。编码中允许灵活调整实现细节，只在方向改变时才需要用户介入。既不僵化，也不放任。

**vs agentic coding**：不靠 AI 自主判断任务完成度。每一步产出都经审查官对账，用户只在关键决策点被通知，其余由审查官代劳。

## 工作流

### Code 模式（改代码）

```
Clarify               Analyze             Review              Code              Commit            Archive
 主 Agent              分析师               方案审查官           程序员             代码审查官          主 Agent
  │                    │                    │                    │                    │                 │
  ├─ 多轮需求追问        ├─ 根因定位            ├─ 七角度对立审查      ├─ 拆任务            ├─ 偏离三层裁决       ├─ 登记索引
  ├─ 写需求文档          ├─ 设计方案            ├─ 通过/驳回           ├─ 编码              ├─ 自试验证          ├─ 清理状态
  │                    ├─ 历史检索            │   ├ 驳回→🔴内循环→analyst  ├─ 编译              ├─ 完成度对账         └─ 待处理回顾
  ▼                    ▼                    │   └ 通过→🟢          ├─ 自测              │   ├ 正常→💾commit
                       [AGENT:COMPLETED]    ▼                    │                    │   ├ 偏离→🔴内循环→coder
                       → 主 Agent 切阶段     👤 用户批准 ──────────►                   │   └ 方向变更→👤用户
                                            │                    ▼                    ▼
                                            │                   [AGENT:COMPLETED]     git commit
                                            │                   → 主 Agent 切阶段      （主 Agent 执行）
                                            ▼
                                            [AGENT:COMPLETED][MODE:lightweight|normal|report]
                                            → 主 Agent 读标记做路由
```

### Clarify 入口（需求梳理 + 深度分析）

Code 和 Analyze 统一从 Clarify 进入，Clarify 由主 Agent 轻量交互式调度（多轮追问，不委派子 agent）。内部根据意图自动分流：

| 子路径 | 触发词 | 产出 | 特点 |
|--------|--------|------|------|
| 修改子路径 | 修/改/加/实现/梳理/方案 | proposal.md | 追问需求 → 代码验证 → 写方案 → 审查 → 用户批准 |
| 分析子路径 | 分析/排查/调研/定位 | analysis.md | 委托分析师 → 审查报告 → 展示（可随时转入修改子路径） |

分析完想改代码无需重来，直接在当前上下文转入修改子路径继续。

### 审查官偏离裁决

AI 编码时不可避免会偏离最初方案。审查官将偏离分为三层处理——执行层偏差（换种写法、加个 import）直接放行；内循环层偏差（多写个文件，等价实现）审查官自行裁决，接受或驳回 coder 在后台修正；只有方向性变更（换架构、加新功能）才上升用户。既不放任，也不唠叨。

## 设计亮点

**多 Agent 协作 + 完成汇报协议**：主 Agent 只负责意图路由、委派子 agent、状态切换和用户硬卡点确认，**绝不直接写代码或读 proposal 正文做技术判断**。各阶段子 Agent 独立执行本职工作，开工校验状态、完工输出结构化完成标记（`[AGENT:COMPLETED]` / `[AGENT:REJECTED:reason]`）。阶段流转由主 Agent 统一管理，子 Agent 绝不自行切换。Clarify、Archive、Weekly 保留在主 Agent 中作为轻量调度。路由信息（轻量/普通模式、自测结果）通过标记的键值扩展传递。

**内循环不惊动状态机**：方案审查官退回去修正属于本阶段内循环，不产生新状态节点。状态机只管硬卡点边界（Stage 间转换），不管 Stage 内迭代。

**程序员不自审偏离**：程序员只管按 proposal 执行，不主动发现偏离。所有偏离由代码审查官在完成度检查时统一发现和裁决——检查者和执行者分离，不自判自案。

**零硬编码工具链**：项目类型、编译命令、部署方式全部探测后写入 project.json，按需加载，跨语言跨框架通用。

## Workspace 结构

```
workspace/
├── projects.md           项目登记簿
└── projects/<id>/
    ├── project.json      工具链 + 项目摘要
    ├── task-tracker.md   活跃任务追踪
    ├── pending-issues.md 待处理问题
    ├── proposals/        方案文档
    ├── tasks/            任务执行计划
    ├── docs/             分析报告 / 调试记录
    └── archive/          已完成归档
        ├── index.md      归档索引（历史检索用）
        └── YYYY-MM/
```

## 换电脑迁移

将 workspace 目录整体拷贝到新电脑，在新电脑上重新安装 AI码律，在 `codeedict-config.json` 中更新 `workspacePath` 指向新位置。所有项目数据、方案历史跟着走。

> `projects.md` 中登记的源码路径在新电脑上需要手动更新。
